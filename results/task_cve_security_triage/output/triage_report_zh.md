# 漏洞分诊报告 — `acme-webapp`

**扫描工具:** AcmeSec Vulnerability Scanner v3.2.1
**扫描日期:** 2026-04-10
**分诊日期:** 2026-04-11
**审核人:** 安全工程团队
**范围:** 涵盖运行时依赖、构建工具链及传递依赖中的 10 个 CVE

## 优先级定义（依据变更管理策略）

| 级别 | SLA | 触发条件 |
|------|-----|---------|
| **P0 — 严重** | 24 小时内部署（允许热修复） | 存在在野利用、未认证的远程代码执行/认证绕过影响运行时路径，或直接影响支付范围 |
| **P1 — 高危** | 7 天内部署 | 高 CVSS 运行时问题且有已验证的利用路径；影响已认证用户或可能暴露敏感数据 |
| **P2 — 中危** | 30 天内部署（与迭代版本批量发布） | 中 CVSS，已被现有控制措施缓解、仅影响构建工具，或需要非默认配置 |
| **P3 — 低危** | 90 天内部署（由团队自行决定） | 低 CVSS，需要罕见前置条件，或属于信息性/纵深防御性质 |

**PCI DSS 范围说明:** 当 CVE 影响的代码路径可到达 `/api/v1/payments/*`（持卡人数据环境边界）、认证/授权中间件或存储支付记录的数据库层时，标记为"在范围内"。仅影响构建工具、从支付代码路径不可达的传递依赖，或需要非默认生产配置的 CVE 视为范围外，但仍按策略进行修复。

## 分诊结果（按严重程度从高到低排列）

### 1. CVE-2026-31845 — `jsonwebtoken`（JWT 签名绕过）
- **CVSS:** 9.1（严重）— `AV:N/AC:L/PR:N/UI:N/S:U/C:H/I:H/A:N`
- **当前版本 → 修复版本:** 8.5.1 → 9.0.1（主版本升级）
- **优先级: P0**
- **理由:** 存在在野活跃利用，可实现对 RS256 签名认证所用 JWT 的未授权伪造。令牌伪造直接危及所有端点（包括 `/api/v1/payments/*`）的认证安全，且无法通过 WAF 规则缓解；需在 24 小时内完成热修复。
- **PCI DSS 范围:** **在范围内**（支付端点的认证绕过；违反要求 6/8 访问控制）。

### 2. CVE-2026-29112 — `express`（通过 Transfer-Encoding 头实现远程代码执行）
- **CVSS:** 9.8（严重）— `AV:N/AC:L/PR:N/UI:N/S:U/C:H/I:H/A:H`
- **当前版本 → 修复版本:** 4.17.1 → 4.19.2（次版本升级）
- **优先级: P0**
- **理由:** 核心 HTTP 服务器上的未认证远程代码执行漏洞，且已有公开 PoC。虽然 ALB/WAF 提供了部分边缘过滤，但 Transfer-Encoding 走私在 Node.js 层处理，可到达 `src/server/api.js` 和 `src/server/middleware.js`（前置于支付路由）；必须与 CVE-2026-31845 在同一热修复窗口发布。
- **PCI DSS 范围:** **在范围内**（支付端点的 HTTP 前端入口；RCE 将危及持卡人数据环境）。

### 3. CVE-2026-27519 — `pg`（通过数组参数绕过的 SQL 注入）
- **CVSS:** 8.1（高危）— `AV:N/AC:L/PR:L/UI:N/S:U/C:H/I:H/A:N`
- **当前版本 → 修复版本:** 8.7.1 → 8.11.4（次版本升级）
- **优先级: P1**
- **理由:** 针对 `user` 和 `order` 模型（支撑支付记录）所使用的 PostgreSQL 驱动的已认证 SQL 注入。参数化查询通常可提供保护，但精心构造的数组参数可绕过转义——已有公开 PoC；需在 7 天窗口内发布。
- **PCI DSS 范围:** **在范围内**（查询包含支付记录的 `order` 模型；违反要求 6.5.1 SQL 注入控制）。

### 4. CVE-2026-30291 — `axios`（通过重定向 URL 解析的 SSRF）
- **CVSS:** 7.4（高危）— `AV:N/AC:H/PR:N/UI:N/S:U/C:H/I:H/A:N`
- **当前版本 → 修复版本:** 0.21.1 → 1.6.8（主版本升级）
- **优先级: P1**
- **理由:** `external-api.js` 和 webhook 处理器中的 SSRF 漏洞；部署文档明确指出 WAF"无法阻止所有 SSRF 向量"，能够控制重定向目标的攻击者可横向移动至内部 RDS/ECS 元数据。利用复杂度较高（需要用户提供的 URL + follow-redirects），但对内部服务的爆炸半径需要 7 天 SLA。
- **PCI DSS 范围:** **在范围内**（webhook 处理器可从支付回调流程到达；SSRF 可能触及持卡人数据网络）。

### 5. CVE-2026-28003 — `lodash`（原型污染 DoS）
- **CVSS:** 7.5（高危）— `AV:N/AC:L/PR:N/UI:N/S:U/C:N/I:N/A:H`
- **当前版本 → 修复版本:** 4.17.20 → 4.17.25（补丁升级）
- **优先级: P2**
- **理由:** CVSS 评分较高但影响仅限于可用性（DoS），通过原型污染实现，利用需要将攻击者控制的 JSON 通过 `_.merge`/`_.defaultsDeep` 传入配置/数据转换——不在支付请求路径中。补丁级别升级风险低，可批量纳入下一迭代发布。
- **PCI DSS 范围:** **范围外**（仅影响可用性；不涉及持卡人数据的机密性/完整性）。

### 6. CVE-2026-25877 — `helmet`（CSP nonce 绕过）
- **CVSS:** 5.4（中危）— `AV:N/AC:L/PR:N/UI:R/S:U/C:L/I:L/A:N`
- **当前版本 → 修复版本:** 4.6.0 → 7.1.0（主版本升级）
- **优先级: P2**
- **理由:** 中等严重程度的 CSP 绕过，需要存在漏洞的浏览器和用户交互（点击/导航）。仅作为 XSS 的纵深防御——不直接暴露数据；helmet 主版本升级需要验证中间件配置。批量纳入迭代发布。
- **PCI DSS 范围:** **范围外**（浏览器端防御；支付端点为 JSON API，不提供 CSP 保护的 HTML）。

### 7. CVE-2026-34201 — `semver`（版本解析中的 ReDoS）
- **CVSS:** 5.3（中危）— `AV:N/AC:L/PR:N/UI:N/S:U/C:N/I:N/A:L`
- **当前版本 → 修复版本:** 5.7.1 → 7.5.4（主版本升级，构建工具）
- **优先级: P2**
- **理由:** 仅影响构建/CI 工具链（非运行时），利用需要攻击者控制的版本字符串到达 semver 解析。主版本升级在 CI 中测试，不影响客户；随迭代发布。
- **PCI DSS 范围:** **范围外**（仅构建依赖；无运行时持卡人数据暴露）。

### 8. CVE-2026-33010 — `minimatch`（glob 解析中的 ReDoS）
- **CVSS:** 5.3（中危）— `AV:N/AC:L/PR:N/UI:N/S:U/C:N/I:N/A:L`
- **当前版本 → 修复版本:** 3.0.4 → 3.1.2（次版本升级，构建工具）
- **优先级: P2**
- **理由:** 仅影响构建的 ReDoS，对可用性影响轻微（CI 作业，非生产环境）。次版本升级，风险低；与其他构建工具修复批量发布。
- **PCI DSS 范围:** **范围外**（仅构建依赖）。

### 9. CVE-2026-37188 — `qs`（查询字符串嵌套对象 DoS）
- **CVSS:** 3.7（低危）— `AV:N/AC:H/PR:N/UI:N/S:U/C:N/I:N/A:L`
- **当前版本 → 修复版本:** 6.5.2 → 6.12.1（次版本，express 的传递依赖）
- **优先级: P3**
- **理由:** 仅在 `depth` 选项被显式提高到默认值以上时可利用；我们的代码使用 express 默认值，且 express 4.19.2（由 P0 修复引入）已将 qs 升级到安全版本。纵深防御性的显式升级可由团队自行安排。
- **PCI DSS 范围:** **范围外**（默认配置阻止利用）。

### 10. CVE-2026-36500 — `debug`（冗余错误信息泄露）
- **CVSS:** 3.1（低危）— `AV:N/AC:H/PR:N/UI:R/S:U/C:L/I:N/A:N`
- **当前版本 → 修复版本:** 2.6.9 → 4.3.5（主版本，传递依赖）
- **优先级: P3**
- **理由:** 需要在生产环境启用 `DEBUG` 环境变量（根据我们的 Fargate 任务定义并通过 IaC 确认，当前未启用）并需要用户交互。传递依赖，实际风险可忽略；在 90 天窗口内由团队自行安排。
- **PCI DSS 范围:** **范围外**（受配置错误约束；生产环境 DEBUG 已禁用）。

## 横切面观察

- **10 个 CVE 中有 4 个在 PCI DSS 范围内**，全部为 P0 或 P1——这些是 6 周后 SOC 2 评估的审计相关项。
- P0 热修复捆绑包（express + jsonwebtoken）必须在**今天/明天**加急处理，独立于 4 月 14 日/17 日的计划部署窗口。
- 3 个 CVE 涉及**主版本升级**（jsonwebtoken、axios、helmet、semver、debug）——这些需要在代码审查和回归测试中格外关注（详见修复计划）。
- `qs` 很可能因 express 升级而被隐式修复，因为 express 4.19.2 依赖 qs ≥6.11；我们将在 P0 补丁后通过 `npm ls qs` 验证。
- `debug` 受生产环境配置约束；建议立即进行配置审计（确认 Fargate 任务定义中未设置 `DEBUG`），作为升级安排期间的零成本缓解措施。
